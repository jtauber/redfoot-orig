Redfoot Version Conventions

Redfoot version numbers are based the system used by the Linux kernel, where there are typically two active branches at any one time, a stable one and a development one. Most people will want to use a stable release (especially for production). Many development releases will be of alpha quality.

Redfoot version numbers come in three forms:

Redfoot X.EVEN.Y
A stable release. e.g. Redfoot 1.0.3
Redfoot X.ODD.Y
A development release. e.g. Redfoot 1.1.7
Redfoot X.ODD.99-preY
A development release that is a candidate to become a stable release. eg Redfoot 1.1.99-pre2
